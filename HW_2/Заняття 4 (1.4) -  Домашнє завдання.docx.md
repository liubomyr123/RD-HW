# **1\. Постановка задачі**

Написати консольну C++ програму, яка виконує **покрокову симуляцію** руху дрона до точки скиду боєприпасу на одну з 5 рухомих цілей. Програма обирає найближчу за часом ціль, враховує зміщення цілей, коригує курс на випередження (lead targeting) і моделює розгін, гальмування та повороти дрона.

Програма базується на ДЗ 1 (балістична задача) і розширює її циклами, масивами, enum та функціями.

# **4\. Масив координат цілей**

Координати 5 цілей зчитуються з файлу **targets.txt** у масиви **targetXInTime\[5\]\[60\]** та **targetYInTime\[5\]\[60\]**. Перший індекс — номер цілі, другий — крок часу. Масив зациклений: після кроку 59 повертаємося до кроку 0\.

Формат targets.txt — 10 рядків: 5 рядків X-координат (60 значень через пробіл), потім 5 рядків Y-координат.

Інтерполяція між кроками — **лінійна**:

int idx \= (int)floor(t / arrayTimeStep) % 60;

int next \= (idx \+ 1\) % 60;

float frac \= (t \- idx \* arrayTimeStep) / arrayTimeStep;

float x \= targetX\[i\]\[idx\] \+ (targetX\[i\]\[next\] \- targetX\[i\]\[idx\]) \* frac;

# **5\. Стани дрона (enum)**

# **6\. Фізична модель**

## **6.1. Прискорення дрона**

Прискорення визначається зі швидкості атаки та шляху розгону:

a \= attackSpeed² / (2 · accelerationPath)

Шлях гальмування \= шлях розгону \= accelerationPath.

## **6.2. Балістична задача**

Формули часу польоту (метод Кардано), горизонтальної дистанції (степеневий ряд до t⁵) та точки скиду — з ДЗ 1 без змін.

## **6.3. Lead targeting (коригування на випередження)**

Балістика розраховується не до поточної позиції цілі, а до **прогнозованої** позиції в момент прильоту дрона до точки скиду.

**Швидкість та напрямок руху цілі** обчислюються з масиву координат через кінцеві різниці:

float dx \= targetX(t \+ dt) \- targetX(t);

float dy \= targetY(t \+ dt) \- targetY(t);

float targetVx \= dx / dt;

float targetVy \= dy / dt;

де dt — малий крок (наприклад, simTimeStep або arrayTimeStep).

**Прогнозована позиція цілі** на момент t \+ totalTime:

float predictedX \= targetX(t) \+ targetVx \* totalTime;

float predictedY \= targetY(t) \+ targetVy \* totalTime;

**Алгоритм на кожному кроці:**

1. Розрахувати орієнтовний час прильоту дрона до точки скиду (totalTime) для поточної позиції цілі

2. Обчислити швидкість цілі (targetVx, targetVy) через кінцеві різниці

3. Інтерполювати прогнозовану позицію цілі на момент currentTime \+ totalTime

4. Перерахувати балістику до прогнозованої позиції

Це повторюється для кожної з 5 цілей, обирається ціль з мінімальним totalTime.

## **6.4. Поворот дрона**

Якщо кут між поточним напрямком і новим напрямком \> turnThreshold:

5. Дрон гальмує (шлях гальмування \= accelerationPath)

6. Повертається на місці. Час повороту \= |deltaAngle| / angularSpeed

7. Розганяється у новому напрямку

Якщо кут ≤ turnThreshold — дрон змінює напрямок без зупинки.

## **6.5. Вибір цілі**

На кожному кроці для кожної цілі розраховується загальний час підльоту до точки скиду. Якщо обрана ціль змінюється, до часу додається час зупинки (timeToStop) залежно від поточного стану дрона:

STOPPED:       timeToStop \= 0

ACCELERATING:  timeToStop \= залишок часу до зупинки

MOVING:        timeToStop \= attackSpeed / acceleration

TURNING:       timeToStop \= залишок часу повороту

# **7\. Алгоритм симуляції**

Основний цикл while працює доки дрон не досягне точки скиду або кількість кроків не перевищить MAX\_STEPS \= 10000\.

**На кожній ітерації:**

8. Інтерполювати позиції всіх 5 цілей

9. Для кожної цілі: розрахувати балістику з lead targeting, точку скиду, орієнтовний час

10. Обрати ціль з мінімальним загальним часом (з врахуванням timeToStop при зміні цілі)

11. Перевірити кут повороту. Якщо \> turnThreshold — змінити стан на DECELERATING/TURNING

12. Оновити координати, швидкість та стан дрона відповідно до поточної фази

13. Записати дані кроку у вихідні масиви

14. currentTime \+= simTimeStep

Симуляція завершується, коли дрон досягне точки скиду (hitRadius) і скине боєприпас.

# **8\. Вихідні дані**

Програма виводить дані у файл **simulation.txt**. Формат:

Рядок 1: N (кількість кроків)

Рядок 2: x0 y0 x1 y1 ... xN yN (координати дрона)

Рядок 3: d0 d1 ... dN (напрямки, рад)

Рядок 4: s0 s1 ... sN (стани: 0-4)

Рядок 5: t0 t1 ... tN (індекси поточної цілі)

MAX\_STEPS \= 10000\. Записується тільки до моменту скиду.
